# Web Spec Template

Use this structure unless the user asks for a shorter or more specific format.

## 1. Project Summary

- Product name
- Product type
- Main objective
- Target users
- Core business outcome

## 2. Assumptions

- List assumptions that were inferred because the request was incomplete.

## 3. User Roles

- Primary role
- Secondary role
- What each role needs to do

## 4. Sitemap

- Page list
- Parent and child relationships if relevant

## 5. Main User Flows

For each major flow, define:
- entry point
- steps
- decision points
- success state
- error or empty states

## 6. Page-by-Page Specification

For each page, define:
- purpose
- target user
- key sections
- key components
- primary CTA
- secondary CTA
- important states

## 7. Frontend Design Direction

- visual tone
- layout character
- color direction
- typography direction
- component style
- motion guidance
- responsive behavior
- accessibility baseline

## 8. Content and Data Needs

- text content needed
- images or illustrations needed
- dynamic data needed
- validation or business rules

## 9. Build Notes

- recommended frontend stack if requested
- implementation risks
- dependencies or integrations

## 10. Reusable Build Prompt

When the user asks for a prompt, end with:
- project context
- requested pages
- layout and visual direction
- responsive and accessibility requirements
- implementation constraints
